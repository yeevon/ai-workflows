**Root cause вЂ” `PlannerPlan` constraint space size.** From [planner.py:87-108](../../../../ai_workflows/workflows/planner.py#L87-L108):

```python
class PlannerStep(BaseModel):
    index: int = Field(ge=1)                                          # numeric bound
    title: str = Field(min_length=1, max_length=200)                  # string bounds
    rationale: str = Field(min_length=1, max_length=1000)             # string bounds
    actions: list[str] = Field(min_length=1, max_length=10)           # array bound + nested string

class PlannerPlan(BaseModel):
    goal: str = Field(min_length=1)                                   # string bound
    summary: str = Field(min_length=1, max_length=1000)               # string bounds
    steps: list[PlannerStep] = Field(min_length=1, max_length=25)     # array bound + nested model
    model_config = {"extra": "forbid"}                                # closed-world
```

Gemini's own error taxonomy (quote above) explicitly calls out all three offenders in our schema: "array length limits" (nested), "complex value matchers" (min/max bounds), and the cumulative string-bound count.

## Additions beyond spec вЂ” audited and justified

### Addition: module-level `PLANNER_RETRY_POLICY` constant (vs. `build_planner`-local variable)

The T07a spec body offered two implementation shapes for the retry-policy pin:

> either expose `policy` for test inspection or, simpler, re-instantiate the constant inside the test and pin the value.

The Builder chose a third shape: extract to a module-level `PLANNER_RETRY_POLICY` constant and have `build_planner` reference it. This is smaller than "expose `policy`" (no accessor function, no getter) and stricter than "re-instantiate in the test" (the test asserts against the *same object* `build_planner` uses, not a literal copy).

**Justified:** Preserves the spec's simplicity preference; gives the pinning test (`test_planner_retry_policy_bumps_transient_attempts_to_five`) a real bind against the production constant; adds no new public surface beyond one symbol to `__all__`; docstring cites KDR-006 and the 503-waste reasoning inline so future readers don't have to chase the commit history for the rationale.

### Addition: retry-policy pin test is a new top-level test (vs. a new assertion in an existing test)

The T07a spec body for AC-7 says:

> add a one-line assertion that `max_transient_attempts == 5`

The Builder wrote it as a new two-line top-level test (`test_planner_retry_policy_bumps_transient_attempts_to_five`) rather than an in-line assertion inside an existing test. This was to keep the assertion scope narrow (retry-policy-only, not coupled to the full happy-path invoke), and to let the docstring cite the 503-waste reasoning in the same place a future grep would land.

**Justified:** Low cost (+1 test count, +9 lines including docstring); high readability; adds no cross-coupling to other tests.

### Addition: CHANGELOG entry under `[Unreleased]` (vs. nested inside the dated M3 section)

The T07a spec body says:

> Under `## [Unreleased]`, add `### Added вЂ” M3 Task 07a: Planner Structured Output (YYYY-MM-DD)`.

The Builder placed the entry at the top of `[Unreleased]`, above the Architecture pivot entry that has been carried since M1. This follows the Keep-a-Changelog reverse-chronological convention within a section and mirrors how M3 T01вЂ“T07 entries were placed during their own Builder cycles.

**Justified:** Matches the pinned convention.
